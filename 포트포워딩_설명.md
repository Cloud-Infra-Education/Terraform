# 포트 포워딩에 대한 설명

## 포트 포워딩이란?

포트 포워딩(Port Forwarding)은 **원격 서버의 포트를 로컬 컴퓨터로 가져오는** 기능입니다.

### 현재 상황

VS Code에서 보이는 포트 포워딩:
- `localhost:8000` → 원격 서버의 8000 포트
- `localhost:7700` → 원격 서버의 7700 포트  
- `localhost:8080` → 원격 서버의 8080 포트

이것은 **로컬 개발용**입니다. 로컬 컴퓨터에서 `http://localhost:8000`으로 접근하면, 원격 서버의 8000 포트로 연결됩니다.

## 프로덕션 접근

프로덕션 환경에서는:
- **EKS 파드에서 실행**: 백엔드는 EKS 워커 노드의 파드에서 실행됩니다
- **ALB를 통한 접근**: `https://api.matchacake.click/api/v1/health`
- **포트 포워딩과 무관**: 프로덕션 접근은 포트 포워딩과 완전히 별개입니다

## 포트 포워딩을 "api.matchacake.click"로 바꿀 수 있나요?

**아니요, 불가능합니다.** 포트 포워딩은:
- 원격 서버의 포트 → 로컬 포트 (단방향)
- 로컬 포트를 원격 주소로 바꿀 수 없음

## 올바른 접근 방법

### 로컬 개발 시
```bash
# 포트 포워딩 사용 (VS Code 자동)
http://localhost:8000/docs
```

### 프로덕션 접근 시
```bash
# ALB를 통한 접근
https://api.matchacake.click/api/v1/health
https://api.matchacake.click/docs  # 백엔드 설정 필요
```

## /docs 경로 문제

`/docs` 경로가 404인 이유:
1. 백엔드 코드에서 Swagger UI가 비활성화되었을 수 있음
2. FastAPI의 `root_path` 설정 문제일 수 있음
3. 백엔드 코드 확인 필요

포트 포워딩과는 **무관**합니다.
